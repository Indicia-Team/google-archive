submit (line 132)

Submits the data by:

  • Calling the preSubmit function to clean data.

  • Linking in any foreign fields specified in the "fk-fields" array.
  • For each entry in the "supermodels" array, calling the submit function for that model and linking in the resultant object.
  • Checking (by a where clause for all set fields) that an existing record does not exist. If it does, return that.
  • Calling the validate method for the "fields" array.
If successful, returns the id of the created/found record. If not, returns null - errors are embedded in the model.

  • access: public
